## 2026-03-19 - FileManager Enumerator Pre-fetching
**Learning:** When using `FileManager.enumerator(at:includingPropertiesForKeys:)`, any property requested later inside the loop via `resourceValues(forKeys:)` MUST be included in the `includingPropertiesForKeys` array. If a key is missing from the pre-fetch array but requested in the loop (e.g., requesting `.isRegularFileKey` while iterating), `URL.resourceValues()` falls back to a synchronous disk access for every single file. In deep cache directories, this transforms a fast O(1) bulk directory read into an O(N) disk I/O bottleneck.
**Action:** Always verify that the keys array passed to `resourceValues(forKeys:)` inside an enumerator loop is a subset of the `includingPropertiesForKeys` passed to `FileManager.enumerator`.
