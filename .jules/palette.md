## 2024-05-18 - Clickable list rows in SwiftUI
**Learning:** By default, SwiftUI lists with embedded buttons can have small hit targets, especially if a button only wraps a small element (like a checkbox) rather than the entire row contents. Furthermore, `HStack`s with empty space will not intercept clicks in the transparent areas without an explicit content shape. Additionally, interactive items need to have their children combined for VoiceOver for an optimal accessibility experience.
**Action:** When creating custom list rows that should act as a single selectable item, wrap the entire row contents (e.g., `HStack`) in a `Button(action:)`, apply `.contentShape(Rectangle())` to the content, and use `.buttonStyle(.plain)` to prevent unintended text styling. For accessibility, always add `.accessibilityElement(children: .combine)` and state traits like `.accessibilityAddTraits(.isSelected)` to the outer button.
