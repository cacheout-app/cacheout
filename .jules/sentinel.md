## 2024-05-18 - Secure Process execution and when to use shell wrappers
**Vulnerability:** Using `/bin/bash -c` with string inputs for external commands poses a command injection risk when user input is involved. However, removing shell wrappers completely breaks commands relying on shell features (like globbing, tilde expansion, or pipelines).
**Learning:** Naively replacing `/bin/bash -c` with `/usr/bin/env` and `.split(" ")` will cause functional regressions for internal app constants that require shell expansion. Only static, explicitly structured commands (like `docker system prune -f`) should be refactored to use direct execution.
**Prevention:** For static commands without shell expansions, set `process.executableURL = URL(fileURLWithPath: "/usr/bin/env")` and pass the tool name and arguments explicitly via `process.arguments`. Replicate shell features like `2>&1` natively by assigning the same `Pipe()` to `standardOutput` and `standardError`. For internal trusted commands requiring shell features, `/bin/bash -c` remains necessary but must never accept unsanitized user input.
