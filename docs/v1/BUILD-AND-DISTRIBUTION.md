# Build and Distribution

## Requirements

- macOS 14 (Sonoma) or later
- Xcode 15+ or Swift 5.9+ toolchain
- For DMG creation: `create-dmg` (`brew install create-dmg`)

## Build Methods

### Build from Source (CLI binary) — helper unavailable

```bash
swift build -c release
```

Output: `.build/release/Cacheout`

This creates a standalone binary that can run in both GUI and CLI mode.
Note: Running the unbundled binary will show SourceKit diagnostics for missing
`Bundle.module` — use the app bundle for production use.

**Helper status:** The privileged helper (`install-helper` / `uninstall-helper`)
is **not available** with this build method. `SMAppService.daemon(plistName:)`
requires the helper binary and LaunchDaemon plist to be embedded at
`Contents/Library/LaunchDaemons/` inside a proper app bundle. The CLI will
report `HELPER_UNAVAILABLE` and the GUI onboarding will show "Helper Unavailable"
— this is expected, not a bug.

### Build & Bundle (.app) — helper unavailable

```bash
bash scripts/bundle.sh
```

This script:
1. Runs `swift build -c release`
2. Creates `Cacheout.app/Contents/` directory structure
3. Copies the binary to `Contents/MacOS/Cacheout`
4. Copies `Info.plist` to `Contents/`
5. Copies the app icon (`Cacheout.icns`) to `Contents/Resources/`
6. Copies Sparkle framework to `Contents/Frameworks/`

The resulting `Cacheout.app` can be dragged to `/Applications`.

**Helper status:** This script does **not** embed the helper binary or
LaunchDaemon plist. Helper install/uninstall will report unavailable.
To produce a helper-capable bundle, use the XcodeGen project (see below).

### XcodeGen / Xcode Project — helper capable

```bash
xcodegen generate   # generates Cacheout.xcodeproj from project.yml
open Cacheout.xcodeproj
```

Build and run from Xcode with Cmd+R. The generated project includes Copy Files
build phases that embed the `CacheoutHelper` binary and
`com.cacheout.memhelper.plist` into `Contents/Library/LaunchDaemons/`, which is
required for `SMAppService.daemon(plistName:)` to work. This is the only local
build method that supports helper registration.

### Open Package.swift in Xcode — helper unavailable

```bash
open Package.swift
```

Build and run from Xcode with Cmd+R. Xcode resolves SPM dependencies automatically.

**Helper status:** Opening `Package.swift` directly uses SwiftPM's build system,
which does not include the Copy Files phases from `project.yml`. The helper will
be unavailable. Use `xcodegen generate` and the generated `.xcodeproj` instead
for helper-capable builds.

## Creating a DMG

```bash
bash scripts/build-dmg.sh
```

Requirements: `create-dmg` (`brew install create-dmg`)

Creates a distributable DMG installer with:
- Custom background image (generated by `scripts/make-dmg-bg.py`)
- App icon and Applications folder alias
- Window size and icon positioning

Output: `dist/Cacheout-1.0.0.dmg`

## Homebrew Installation — helper capable

A Homebrew cask is available at `homebrew/cacheout.rb`:

```bash
brew install --cask cacheout
```

The cask installs `Cacheout.app` (with embedded helper) and exposes the CLI
binary as `cacheout`. This is the recommended distribution method and the only
path that supports helper registration out of the box.

The helper is automatically unregistered on `brew uninstall --cask cacheout`
via the cask's `uninstall_preflight` hook.

## Project Configuration

### Package.swift

```swift
// swift-tools-version: 5.9
let package = Package(
    name: "Cacheout",
    platforms: [.macOS(.v14)],
    dependencies: [
        .package(url: "https://github.com/sparkle-project/Sparkle", from: "2.6.0"),
    ],
    targets: [
        .executableTarget(
            name: "Cacheout",
            dependencies: ["Sparkle"],
            path: "Sources/Cacheout",
            resources: [.process("Resources")]
        )
    ]
)
```

### Info.plist

| Key | Value | Description |
|-----|-------|-------------|
| `CFBundleIdentifier` | `com.cacheout.app` | Bundle identifier |
| `CFBundleVersion` | `1` | Build number |
| `CFBundleShortVersionString` | `1.0.0` | Display version |
| `LSMinimumSystemVersion` | `14.0` | Minimum macOS version |
| `LSApplicationCategoryType` | `public.app-category.utilities` | App Store category |
| `LSUIElement` | `false` | Show in Dock |

### VERSION File

The `VERSION` file at the project root contains the current version string (`1.0.0`).

## Watchdog (Background Monitoring)

The `Watchdog/` directory contains scripts for optional background monitoring:

| File | Purpose |
|------|---------|
| `cacheout-watchdog.sh` | Shell script that runs Cacheout scans periodically |
| `com.cacheout.watchdog.plist` | LaunchAgent plist for macOS scheduling |
| `install-watchdog.sh` | Installation script for the LaunchAgent |

### Installing the Watchdog

```bash
bash Watchdog/install-watchdog.sh
```

This copies the LaunchAgent plist to `~/Library/LaunchAgents/` and loads it.

### Removing the Watchdog

```bash
launchctl unload ~/Library/LaunchAgents/com.cacheout.watchdog.plist
rm ~/Library/LaunchAgents/com.cacheout.watchdog.plist
```
